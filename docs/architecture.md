# Qdrant 向量数据库架构设计文档

## 1. 项目概述

Qdrant 是一个高性能的向量相似性搜索引擎和向量数据库，使用 Rust 语言编写。它提供了一个生产就绪的服务，具有便捷的 API 来存储、搜索和管理点（带有附加负载的向量）。Qdrant 专为扩展过滤支持而设计，使其在神经网络或语义匹配、分面搜索和其他 AI 应用中非常有用。

## 2. 架构概览

Qdrant 采用模块化架构，主要由以下几个核心库组成：

- `api`: REST 和 gRPC API 实现
- `collection`: 集合管理和操作
- `common`: 共享工具和通用功能
- `segment`: 核心向量存储和索引
- `shard`: 分片和分布式逻辑
- `storage`: 存储管理和持久化
- `gpu`: GPU 加速组件
- `sparse`: 稀疏向量支持

## 3. 核心组件详解

### 3.1 API 层

API 层提供 REST 和 gRPC 接口，分为两个主要部分：

- **REST API**: 通过 Actix Web 框架实现，提供 HTTP 接口
- **gRPC API**: 通过 Tonic 框架实现，提供高性能的 RPC 接口

API 层负责处理客户端请求，将请求转换为内部操作，并返回结果。

### 3.2 集合管理 (Collection)

Collection 模块是 Qdrant 的核心组件之一，负责：

- 集合的创建、删除和管理
- 集合级别的操作（如搜索、更新、删除）
- 分片管理
- 优化器构建和管理
- 推荐和发现功能

### 3.3 分段存储 (Segment)

Segment 模块负责底层的向量存储和索引：

- **向量存储**: 实现向量的存储和检索
- **索引**: 实现 HNSW 等高效索引结构
- **ID 跟踪**: 管理向量 ID 和其在存储中的位置
- **有效载荷存储**: 存储与向量关联的元数据
- **空间计算**: 实现各种距离计算方法

### 3.4 分片管理 (Shard)

Shard 模块处理数据的分片：

- **分片持有者**: 管理多个分段
- **代理分段**: 在分片迁移期间提供代理功能
- **操作处理**: 处理分片级别的操作
- **查询处理**: 协调分片间的查询操作

### 3.5 存储管理 (Storage)

Storage 模块提供服务层功能，抽象了外部接口：

- **内容管理**: 管理集合元数据和状态
- **分发器**: 根据请求类型将操作分发到适当的组件
- **权限控制**: 实现基于角色的访问控制 (RBAC)

### 3.6 分布式一致性 (Consensus)

Consensus 模块实现分布式一致性，基于 Raft 算法：

- **Raft 节点**: 管理 Raft 状态和操作
- **消息代理**: 处理节点间的消息传递
- **配置变更**: 管理集群成员变更
- **快照管理**: 处理状态快照和恢复

## 4. 分布式架构

### 4.1 集群模式

Qdrant 支持分布式部署，通过以下机制实现：

- **分片**: 数据在多个节点间分片存储
- **复制**: 每个分片可以有多个副本以提高可用性
- **一致性**: 使用 Raft 算法保证数据一致性
- **负载均衡**: 请求在集群节点间分发

### 4.2 节点通信

节点间通信通过 gRPC 实现：

- **P2P 通信**: 节点间直接通信
- **消息传递**: Raft 消息和其他集群消息
- **TLS 支持**: 支持加密通信

### 4.3 一致性保证

- **Raft 算法**: 保证分布式一致性
- **日志复制**: 操作日志在节点间复制
- **领导者选举**: 自动选举集群领导者
- **故障恢复**: 自动处理节点故障

## 5. 数据模型

### 5.1 点 (Point)

Qdrant 中的基本数据单元：

- **ID**: 点的唯一标识符
- **向量**: 高维向量数据
- **有效载荷**: 与点关联的 JSON 元数据

### 5.2 集合 (Collection)

包含点的逻辑容器：

- **分片**: 集合可以分为多个分片
- **配置**: 包含向量维度、距离函数等配置
- **索引**: 为高效搜索建立的索引结构

## 6. 索引和搜索

### 6.1 HNSW 索引

- **高效搜索**: 实现近似最近邻搜索
- **可配置参数**: 支持调整 M、efConstruct 等参数
- **内存/磁盘**: 支持内存和磁盘存储

### 6.2 过滤和有效载荷

- **JSON 有效载荷**: 存储结构化元数据
- **过滤查询**: 支持基于有效载荷的复杂过滤
- **索引优化**: 为过滤字段建立索引

## 7. 性能优化

### 7.1 内存管理

- **jemalloc**: 使用 jemalloc 作为全局分配器
- **内存映射**: 使用内存映射文件优化 I/O
- **向量量化**: 支持向量量化以减少内存使用

### 7.2 硬件加速

- **SIMD**: 支持 SIMD 指令加速
- **GPU**: 可选的 GPU 加速支持
- **并行处理**: 多线程并行搜索和更新

### 7.3 优化器

- **段优化**: 自动优化段结构
- **资源预算**: 控制优化过程的 CPU 和 I/O 预算
- **性能调优**: 可配置的性能参数

## 8. 安全性

- **TLS 加密**: 支持客户端和集群间通信加密
- **API 密钥**: 支持 API 密钥认证
- **JWT RBAC**: 基于 JWT 的基于角色的访问控制
- **CORS**: 可配置的跨域资源共享设置

## 9. 监控和遥测

- **Prometheus 指标**: 提供 Prometheus 格式的指标
- **请求分析**: 可选的遥测报告
- **性能分析**: 请求性能分析功能
- **硬件报告**: 实验性的硬件利用率报告

## 10. 恢复和维护

- **预写日志 (WAL)**: 确保数据持久性
- **快照机制**: 支持数据快照和恢复
- **自动优化**: 自动优化段
- **恢复模式**: 在故障情况下的恢复模式支持

## 11. 配置管理

Qdrant 使用分层配置系统：

1. 编译时默认值（嵌入二进制文件）
2. 主配置文件 (`config/config.yaml`)
3. 环境特定配置 (`config/{RUN_MODE}.yaml`)
4. 本地配置 (`config/local.yaml`)
5. 通过 `--config-path` 的自定义配置
6. 环境变量（前缀为 `QDRANT_`）

## 12. 扩展性

Qdrant 设计为高度可扩展：

- **水平扩展**: 通过分片和复制支持水平扩展
- **插件架构**: 模块化设计支持功能扩展
- **多种 API**: 支持 REST 和 gRPC 接口
- **多客户端**: 提供多种语言的客户端库
