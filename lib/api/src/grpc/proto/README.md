## lib\api\src\grpc\proto 目录作用分析
根据我的分析， lib\api\src\grpc\proto 目录在 Qdrant 项目中扮演着 gRPC API 契约定义中心 的关键角色。以下是详细说明：

### 📁 目录内容
该目录包含 14 个 .proto 文件 ，每个文件定义不同功能的 gRPC 服务接口：

- 核心服务文件 ：
  
  - qdrant.proto - 主入口文件，导入其他 proto 文件
  - qdrant_common.proto - 通用消息定义
  - collections.proto - 集合管理相关（集合创建、配置、更新等）
  - points.proto - 点数据操作（增删改查、向量等）
  - health_check.proto - 健康检查服务
- 内部服务文件 ：
  
  - collections_internal_service.proto - 内部集合服务
  - points_internal_service.proto - 内部点服务
  - qdrant_internal_service.proto - 内部主服务
  - raft_service.proto - Raft 共识协议服务
- 运维服务文件 ：
  
  - snapshots_service.proto - 快照管理服务
  - shard_snapshots_service.proto - 分片快照服务

### 🔄 工作流程
这个目录的工作机制如下：

```
proto 文件定义 → tonic_build 编译 → 
Rust 代码生成
```

在 lib\api\build.rs 中配置了构建流程：

1. 使用 tonic_build::compile() 编译 qdrant.proto 入口文件
2. 自动搜索 proto 目录下的所有依赖文件
3. 生成 Rust 代码保存到 src/grpc/qdrant.rs （标注为 @generated by prost-build ）
4. 添加验证规则和其他自定义属性

### 🎯 主要作用
1. API 契约定义 ：使用 Protocol Buffer 语言定义所有 gRPC 服务的消息格式（Message）和远程过程调用接口（RPC）
2. 跨语言支持 ：proto 文件可以编译成多种语言（Python、Java、C# 等）的客户端库
3. 代码生成基础 ：为 gRPC 客户端/服务器自动生成序列化、反序列化和网络传输代码
4. 版本控制 ：作为单一事实来源，所有 gRPC API 的变更都应该反映在这里
